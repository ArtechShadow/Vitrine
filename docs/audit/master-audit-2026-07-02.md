## 3. LichtFeld deployment answer

**Yes, we were already using LichtFeld, and no, it was never a sidecar.** The planned and actual model is a host-compiled binary bind-mounted read-only at `/opt/gaussian-toolkit/build` (compose:90-91), and the dreamlab 2026-06-21 training genuinely ran LichtFeld v0.4.2-480-gdcd2b047 (fork build) inside the `gaussian-toolkit` container from that mount; the `lichtfeld-studio` service in `docker/docker-compose.yml` is upstream dev-container residue, not a deployed sidecar. That binary vanished ~2026-06-22 (host `./build` is now empty and root-owned), and ADR-021's "image builds from vendor/" was never implemented — Dockerfile.consolidated:131-139 still clones the old jjohare fork with the build failure masked by `|| true`, so today no run can invoke the configured binary (config.py:113). **Restoration options:** (a) host build of the vendor submodule into `./build` (`cmake -S vendor/lichtfeld-studio -B build`, sm_89, cmake≥3.30) — fastest, matches the compose contract; (b) finish the in-image vendor build (the untracked `lf_build.sh`/`lf_build2.sh` are exactly this rewire; currently stuck on vcpkg sdl3) and drop the `|| true`; either way, amend BOUNDARIES.md:26,60 + ADR-021:27 to match whichever lands.

## 4. ArtiFixer verdict

The cleaner-splats hypothesis is **(a)-leaning nothing for the dreamlab room**: repair is opacity-gated with no knobs (z_mix, pipeline_base.py:102-116), its training corruptions are sparse-view-only ("floater" 0 hits, no deblur logic), upstream issue #10 confirms diffusion "barely works" on dense scenes, and the validated 48GB config (ds4) runs diffusion at 1/4 resolution — guaranteed softening, never deblurring. The **one untested lever** is data-side attenuation of the opacity PNGs (s=0.7 / 0.4) to force stronger regeneration, with realistic best-case being floater/mush cleanup at ds4 resolution; note distill is a from-scratch 3DGRUT MCMC retrain, not a filter of our splat. **Recommended cheap-experiment order: (1) Difix3D+ first** — unconditional, light, purpose-built "cleaner splats"; (2) the opacity-attenuation ArtiFixer trial on the already-enabled 48GB-Ada fork; (3) DeblurNVS (2606.01315) only if blur-targeted repair is ever pursued in-model. The blur fix itself remains **recapture** (4K60 short-shutter), full stop.
