What it is

In one sentence.

Point Vitrine at video of a room and its contents; it produces a UE 5.8 scene — the room plus individually reconstructed, correctly-placed, textured object meshes.

room: real splat &/or clean mesh objects: game-style FBX, baked materials optional .ksplat for web
The problem

Real captures fail in different ways.

There is no universal "best path" — the best path is a function of the data.

Motion blur

Hand-held capture smears detail; naïve training bakes the blur into the surface.

Sparse coverage

Too few views in places — geometry collapses to noise or floaters.

Holes

Unobserved regions leave gaps a single fixed pipeline can't fill.

No depth

Commodity phone video, no LiDAR — structure inferred from motion alone.

Why it's hard

Why a single fixed pipeline breaks.

One config can't be optimal across blur, sparsity, holes and missing depth at the same time.

  • Naïve training bakes blur into the surface.
  • Sparse views collapse geometry into floaters.
  • Unobserved regions leave gaps a fixed path can't fill.
  • Net result: brittle, capture-dependent quality.
A pipeline tuned for one failure mode breaks on the next. The fix isn't a better single path — it's choosing the path per capture.
The core idea

Capture-adaptive, multi-pipeline routing.

Diagnose each capture and route it to the reconstruction path that best fits its bottleneck.

  • Vitrine is not a single fixed pipeline.
  • A menu of validated components + capture-conditional enhancers.
  • The router selects the config from the capture profile.

Deep dive

Diagnose & QA, in detail.

Frame quality and structure are measured before anything is reconstructed.

Frame quality

MUSIQ no-reference image-quality scoring + full-res Laplacian sharpest-per-window frame selection.

Structure

COLMAP SfM with ALIKED features + LightGlue matching.

Coverage

View distribution analysis flags under-observed regions.

Hardware

Available GPU / VRAM profile feeds the router's config choice.

Conditional tools

Capture-conditional enhancers.

Extra tools fire only when the capture needs them — gated, evidence-driven, not a fixed post-process.

ArtiFixer

Regenerates floaters / holes / unobserved regions. Does not deblur.

Deblur

Fires for motion-blurred captures.

Densification

Fires for sparse coverage.

Where it's heading

An internal agent controller owns the run.

Vitrine is moving from scripted stages to an agent that drives the pipeline end-to-end.

Select

Diagnose the capture, pick the config / router policy.

Drive

Operate each tool through its control surface.

Evaluate

In-flight QA renders, MUSIQ, mesh/splat checks.

Recover

Restart-resilient lifecycles · phased VRAM.

Plus fan-out: sub-agent meshes choose SOTA components and adversarially verify results before committing.

Architecture principle

LichtFeld is a tool, not the trunk.

Vitrine vendors LichtFeld Studio as a pinned dependency for native 3DGS. Vitrine calls it — it is no longer a fork.

  • Never modified — updates are a submodule tag bump.
  • Its MCP server is the primary 3DGS interface.
  • UE delivery uses our mesh/splat → FBX/NanoGS path.

The honest constraint

The dominant bottleneck is capture quality.

Motion blur + partial coverage — not mesh resolution — sets the ceiling.

  • Finer voxels render the noise more sharply, not cleaner.
  • The binding constraint is the capture, not the algorithm.
  • The fix lives at capture time (docs/capture-methodology.md).
  • ArtiFixer recon-enhancement on trial — verdict pending, gated.
